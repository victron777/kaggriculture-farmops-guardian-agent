## Step 1. Create the public codebase

1. Go to GitHub.
2. Create a new public repository named `kaggriculture-farmops-guardian-agent`.
3. Upload all files from this ZIP.
4. Copy the GitHub repository link.

Recommended repository description:

Autonomous Kaggriculture farming resource-management agent using ADK-style multi-agent design, MCP tools, guardrails, memory, evaluation, and deployable API structure.

## Step 2. Upload the notebook to Kaggle

1. Go to Kaggle.
2. Click **Create** then **New Notebook**.
3. Upload `FarmOps_Guardian_Submission_Notebook.ipynb`.
4. Run all cells.
5. Confirm the evaluation table and trace table appear.
6. Save version.
7. Set visibility to public.
8. Copy the Kaggle notebook link.

## Step 3. Record the demo video

1. Open the Kaggle notebook.
2. Open `VIDEO_SCRIPT.md`.
3. Record your screen for 2 to 5 minutes.
4. Show:
   - project title,
   - architecture section,
   - code cells,
   - scenario output,
   - guardrail example,
   - evaluation result,
   - trace log.
5. Upload the video to YouTube as public or unlisted.
6. Copy the video link.

## Step 4. Submit the Kaggle writeup

1. Open the capstone competition page.
2. Go to **Submit / New Writeup**.
3. Title:

Kaggriculture FarmOps Guardian Agent: ADK-Style Multi-Agent Farm Resource Manager

4. Paste the content from `KAGGLE_WRITEUP.md`.
5. Replace the three placeholders at the bottom:
   - Code repository
   - Kaggle notebook
   - Demo video
6. Submit the writeup.

## Step 5. Final validation before submit

Make sure your writeup has:

- a clear problem statement,
- agent architecture,
- at least three course concepts,
- public codebase link,
- public Kaggle notebook or project link,
- public or unlisted video link,
- no private ABC information,
- evidence of guardrails,
- evidence of evaluation,
- evidence of deployability.
